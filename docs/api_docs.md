# REST API Documentation

A REST API that accepts free-text radiology FINDINGS and returns a concise IMPRESSION summary generated by a fine-tuned BART-base model.

**URL:** `https://radiology-summarizer.onrender.com`
*(Replace with your actual deployed URL)*

---

## POST /summarize

Summarizes radiology report findings into a concise impression.

**Method:** POST

**Authentication required:** NO

**Content-Type:** application/json

### Data Example

```json
{
  "findings": "The trachea is midline. The cardiomediastinal silhouette is normal. The lungs are clear, without evidence of acute infiltrate or effusion. There is no pneumothorax. The visualized bony structures reveal no acute abnormalities."
}
```

### Success Response

**Code:** 200

**Content example:**

```json
{
  "impression": "No acute cardiopulmonary disease."
}
```

### Error Responses

**Condition:** The `findings` field is missing from the request body, or is an empty/blank string.

**Code:** 400

**Content:**

```json
{
  "detail": "findings must not be empty"
}
```

---

**Condition:** Request body is malformed or `findings` is not a string.

**Code:** 422

**Content:**

```json
{
  "detail": [
    {
      "type": "missing",
      "loc": ["body", "findings"],
      "msg": "Field required",
      "input": {}
    }
  ]
}
```

---

**Condition:** Internal server error during model inference.

**Code:** 500

**Content:**

```json
{
  "detail": "Internal server error during inference."
}
```

---

## GET /health

Returns the service health status and whether the model has loaded successfully.

**Method:** GET

**Authentication required:** NO

### Success Response

**Code:** 200

**Content example:**

```json
{
  "status": "ok",
  "model_loaded": true
}
```

---

## Notes

- **Latency:** Inference on CPU takes approximately 2–8 seconds per request depending on findings length. GPU-accelerated deployments reduce this to under 1 second.
- **Input limit:** Findings text longer than 512 tokens will be automatically truncated.
- **Interactive docs:** Available at `https://radiology-summarizer.onrender.com/docs`

---

## Example Usage

### curl

```bash
curl -X POST https://radiology-summarizer.onrender.com/summarize \
  -H "Content-Type: application/json" \
  -d '{"findings": "The lungs are clear. Heart size and mediastinal contours are normal. No osseous abnormalities."}'
```

### Python (requests)

```python
import requests

response = requests.post(
    "https://radiology-summarizer.onrender.com/summarize",
    json={"findings": "The lungs are clear. Heart size and mediastinal contours are normal. No osseous abnormalities."}
)
print(response.json())
# {"impression": "Normal chest radiograph."}
```
